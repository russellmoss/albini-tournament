# /council — Cross-LLM Adversarial Review

> Generated by rstack for **albini-tournament** ()

Send the current implementation plan to two independent LLM reviewers for adversarial review. Each reviewer gets a tailored prompt focused on their strengths.

---

## Step 1: Verify MCP Server

Check if `council-mcp` is available by looking for `ask_codex` and `ask_gemini` tools.

If **not available**, tell the user:
```
Council review requires the council-of-models-mcp server.

To set up:
1. Add API keys to your .env:
   OPENAI_API_KEY=sk-...    (get from https://platform.openai.com/api-keys)
   GEMINI_API_KEY=AI...     (get from https://aistudio.google.com/apikey)

2. Register the MCP server:
   claude mcp add --scope user council-mcp -- npx council-mcp

3. Re-run /council
```
Then STOP.

---

## Step 2: Find and Read Documents

**Required** (at least one must exist):
- `agentic_implementation_guide.md` (or any file matching `*implementation*guide*.md` or `*build*guide*.md`)

**Supporting** (read if they exist):

## Step 3: Investigate the Project

Silently scan the project to provide context to reviewers:
- List top-level directory structure
- Read `package.json` / project config
- List source file structure (just file names, don't read contents)

Do NOT read `.env` files or `node_modules/`.

## Step 4: Send Review Prompts

Send two prompts **in parallel** using council-mcp tools:

### Prompt A → Codex (OpenAI)

Focus areas: Type safety, construction sites, phase ordering, missing edge cases.

```
You are reviewing an implementation plan for albini-tournament ().

Review the attached plan for:
Type safety, construction sites, phase ordering, missing edge cases.

For each issue found, classify as:
- **CRITICAL** — Will cause bugs, data loss, or build failures. Must fix before building.
- **SHOULD FIX** — Correctness or quality issue. Should fix but won't block.
- **DESIGN QUESTION** — Ambiguous decision that needs human input.

Be specific: cite file paths, line numbers, and exact code when possible.
```

Append the FULL TEXT of all documents found in Step 2. Do not summarize — reviewers need complete context.

### Prompt B → Gemini

Focus areas: Business logic correctness, data quality, pattern consistency, integration completeness.

## Step 5: Synthesize Feedback

Write `council-feedback.md` with:

1. **Critical Issues** — Merged and deduplicated from both reviewers
2. **Should-Fix Items** — Merged and deduplicated
3. **Design Questions** — Merged and deduplicated
4. **Suggested Improvements** — Ranked by impact
5. **Raw Responses** — Full text from each reviewer (for reference)

## Step 6: Present and Stop

Tell the user:
- How many critical issues, should-fix items, and design questions were found
- A brief summary of the most important findings
- "Run `/refine` to triage and apply this feedback to your implementation guide."
